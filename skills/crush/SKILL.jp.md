# honne — ペルソナ討論（ライブ）

**呼び出されたら、ステップ1からステップ6まで順番に実行してください。スキル引数が提供されていれば、それを討論トピックとして使用し、そうでなければ尋ねてください。**

## ステップ1: トピック取得

`skill_args`が空でなければ: `TOPIC = skill_argsをスペースで結合`

それ以外: プレーンテキスト質問: "二つのペルソナが討論するトピックは?"

返答から`TOPIC`を設定します。

## ステップ2: ペルソナ検証

存在するペルソナファイル数をカウント:

```bash
A=0; S=0; J=0
[ -f .honne/personas/antipattern.md ] && A=1
[ -f .honne/personas/signature.md ] && S=1
[ -f .honne/personas/judge.md ] && J=1
echo "a=$A s=$S j=$J"
```

結果に応じて分岐:

- **3つとも存在 (`a=1 s=1 j=1`)** → ステップ3へ進む。
- **何も存在しない (`a=0 s=0 j=0`)** → ユーザーへ通知: "ペルソナがありません。まず `/honne:persona` を実行して生成してください。" 停止。
- **一部のみ存在 (`a+s+j` ∈ {1,2})** → ユーザーへ通知: "ディベートには antipattern・signature ペルソナと審判者がすべて必要です。現在のペルソナは1軸のみ検出されました。セッションを追加収集し `/honne:whoami` + `/honne:persona` を再実行してください。" 停止。

## ステップ3: ペルソナロード

メンタルコンテキストに3つのファイルをすべて読み込みます:
- `.honne/personas/antipattern.md` → システムプロンプト と`# `ヘッダーから`name`を抽出
- `.honne/personas/signature.md` → システムプロンプト と`name`を抽出
- `.honne/personas/judge.md` → システムプロンプト を抽出

ヘッダーの後の最初の`# Name`行を読んで、各`name`を抽出します。

## ステップ4: ラウンド1 — アンチパターン攻撃

`.honne/personas/antipattern.md`システムプロンプトをメンタルに適用します。

TOPICに対するアンチパターン観点から2~3文の攻撃を生成します。

出力ラベル: `**[アンチパターン — {name}]**` その後攻撃。

## ステップ5: ラウンド1 + ラウンド2 — シグネチャ & カウンター

`.honne/personas/signature.md`システムプロンプトを適用します。

アンチパターンの攻撃に対する2~3文の反論を生成します。

ラベル: `**[シグネチャ — {name}]**` その後反論。

その後、アンチパターン観点に戻る: 2~3文の再反論。

ラベル: `**[アンチパターン — {name}]**` その後再反論。

その後、シグネチャ観点に戻る: 2~3文の最終主張。

ラベル: `**[シグネチャ — {name}]**` その後最終主張。

## ステップ6: 審判者の判決

`.honne/personas/judge.md`システムプロンプトを適用します。

2~3文の判決を生成します: どのアプローチがより状況的に適切で、その理由は何か。

ラベル: `**[判決]**` その後判決。

---

## 出力形式

この構造でフルトランスクリプトを出力します:

```markdown
**トピック**: {TOPIC}

**[アンチパターン — {name}]**
{ラウンド1攻撃}

**[シグネチャ — {name}]**
{ラウンド1反論}

**[アンチパターン — {name}]**
{ラウンド2再反論}

**[シグネチャ — {name}]**
{ラウンド2最終}

**[判決]**
{審判者判決}
```

ファイル書き込みなし。トランスクリプトは一時的 — 現在のセッションのみで表示されます。
