## 1. Problem Definition

### 1.1 Current State (As-Is)

`/honne:whoami` global scope 실행(604 transcript 파일) 시 다음 문제가 실측됨:

- **scan-transcripts.sh 사실상 실패**: 메시지 한 줄마다 subprocess 4~5개(jq ×4 + python3 ×1) fork → 604 × ~30 msg ≈ **10만 회 fork**. 추가로 `SESSIONS_JSON=$(echo ... | jq '. + [$s]')` 누적 방식이 **O(n²) 문자열 재작성** → 결국 `jq: Argument list too long` (ARG_MAX 초과)로 조용히 종료. 에이전트가 중간에 `.honne/scan_py.py`를 ad-hoc으로 작성해 우회함.
- **extract 스크립트 계약 drift**: `extract-lexicon.sh`가 `--cache` 플래그를 `unknown arg`로 거절. 에이전트가 `.honne/analyze.py`로 6축 분석을 단일 Python 스크립트로 대체 — 계약이 안 맞다는 실증.
- **권한 prompt 반복**: 매 Bash 호출마다 approval. `sleep 30 && cat ...` 폴링은 harness에서 block. run-in-background + Monitor 가이드가 SKILL.md에 명시되어 있지 않음.

### 1.2 Target State (To-Be)

| 지표 | 현재 | 목표 |
|---|---|---|
| 604 파일 scan 시간 | 실패(>10분 후 ARG_MAX) | < 60초 |
| 스크립트 호출 계약 | 스크립트마다 다름 | 공통 base args (`--scope/--since/--cache/--index-ref/--base-dir`) |
| /honne:whoami 1회 실행당 permission prompt | 매 Bash (~20회+) | ≤ 3회 (pre-declared allowlist) |
| bats + pytest 테스트 | 통과 | 통과 (회귀 0) |
| jq 의존성 | 필수 | 제거 |

### 1.3 Business/Technical Impact

honne의 핵심 entrypoint(`whoami`)가 global scope에서 동작 불가 → 플러그인이 사실상 "repo scope 전용". 공개 marketplace 플러그인으로서 첫인상이 망가짐. 계약 drift는 향후 skill 추가 시 ad-hoc Python 스크립트가 계속 생길 조짐.

## 2. Design Decision

### Alternative A: All-Python rewrite

- **Approach**: 모든 `.sh`를 `.py`로 포팅. bats 테스트도 pytest로 통합.
- **Pros**: 단일 언어, subprocess 최소화, 계약 통일 자연스러움.
- **Cons**: 가장 큰 diff. bats 테스트 대거 재작성. pre-commit처럼 shell이 자연스러운 작업도 Python화.
- **Suitable situation**: 장기적으로 Python 패키지로 배포할 때.

### Alternative B: 병목만 Python 교체, 나머지 bash 유지

- **Approach**: `scan-transcripts.sh`와 `extract-*`만 Python으로 재작성. 나머지 6개 bash 유지.
- **Pros**: 최소 diff. 체감 성능 100% 해결.
- **Cons**: 계약 drift 근본 원인 미해결. 두 언어 유지비용. 권한 allowlist 분산.
- **Suitable situation**: 단기 핫픽스 목적.

### Alternative C: 단일 Python entrypoint + bash shim (선정)

- **Approach**: `scripts/honne_py/` Python 패키지로 모든 로직을 통합. `python3 -m honne_py <cmd>` 단일 CLI가 argparse로 공통 base args 강제. 기존 `*.sh`는 1~3줄 shim(`exec python3 -m honne_py <cmd> "$@"`)으로 축소. `_redact.py`/`_tokenize.py`는 package 내부로 흡수.
- **Pros**:
  - argparse 공통 parent parser로 **계약 drift 구조적 차단**
  - 공유 코드(JSONL 스트리밍 파서, redact, cache I/O) 단일 모듈
  - 권한 allowlist가 `Bash(python3 -m honne_py:*)` 하나로 집약 → prompt ≤ 3회
  - bats 테스트는 shim을 그대로 호출하므로 **회귀 보호 유지**
  - SKILL.md 호출부(`bash "$HONNE_ROOT/scripts/foo.sh" ...`)가 **변경 없음**
- **Cons**: 초기 패키지 레이아웃 설계 비용. `__main__.py` + 서브커맨드 라우팅 추가.
- **Suitable situation**: 계약 drift가 실증된 + 전체 리팩터 요청 = **현재 상황**.

### Selection: **Alternative C**

- **Rationale**: (1) 에이전트가 이미 `.honne/analyze.py`를 ad-hoc으로 작성한 것 자체가 "bash 계약이 신뢰 불가"의 실증. (2) scan 실패는 per-message subprocess fork + O(n²) 누적이 원인 — 단일 Python 프로세스 스트리밍이면 수십 초로 종료. (3) bats 테스트 자산 유지하면서 Python 이점 흡수 가능한 유일한 대안. (4) 권한 allowlist 집약 효과는 C만 달성.
- **Rejected**:
  - A: bats 투자(`tests/`) 폐기 비용 과다.
  - B: 계약 drift 근본 원인 미해결 — 3개월 내 동일 문제 재발 예상.
- **Conditions under which this decision is valid**:
  - python3 ≥ 3.9 보장 (Q1에서 hard req 확정)
  - honne이 계속 Claude Code plugin 형태(단일 tree) — PyPI 패키지화하면 A로 재검토
  - transcript 포맷(`.jsonl` with `.message.role`, `.message.content`)이 현재 구조 유지

## 3. Technical Design

### 3.1 Architecture Overview

```
honne/
├── scripts/
│   ├── honne_py/                    # NEW — 단일 Python 패키지
│   │   ├── __init__.py
│   │   ├── __main__.py              # 서브커맨드 dispatch
│   │   ├── cli.py                   # argparse parent + common args
│   │   ├── io.py                    # cache read/write, atomic tmp swap
│   │   ├── scan.py                  # transcript JSONL 스트리밍 파서
│   │   ├── redact.py                # _redact.py 이식
│   │   ├── tokenize.py              # _tokenize.py 이식
│   │   ├── extract.py               # lexicon/obsession/reaction/... 6축
│   │   ├── detect_recurrence.py
│   │   ├── evidence.py
│   │   ├── index.py
│   │   ├── query.py
│   │   ├── record.py
│   │   ├── purge.py
│   │   └── precommit.py
│   ├── scan-transcripts.sh          # shim: exec python3 -m honne_py scan "$@"
│   ├── extract-lexicon.sh           # shim
│   ├── detect-recurrence.sh         # shim
│   ├── evidence-gather.sh           # shim
│   ├── index-session.sh             # shim
│   ├── query-assets.sh              # shim
│   ├── record-claim.sh              # shim
│   ├── purge.sh                     # shim
│   └── pre-commit.sh                # shim
├── skills/{whoami,compare,lexi}/SKILL.md  # Monitor-based guidance 추가
├── tests/{bats,pytest 기존 유지}
└── .claude/settings.json            # permissions.allow 집약 선언
```

**데이터 흐름** (whoami 기준):
```
SKILL.md
  → scan-transcripts.sh [shim]
    → python3 -m honne_py scan --scope global --since ... --cache .honne/cache/scan.json
       → scan.py: find + JSONL stream + redact + cache write (단일 프로세스)
  → extract-lexicon.sh [shim] → python3 -m honne_py extract lexicon --cache ...
  → record-claim.sh [shim] → python3 -m honne_py record claim ...
```

### 3.2 Core Data Model

```python
# honne_py/models.py
from dataclasses import dataclass
from typing import Literal

@dataclass
class Message:
    role: Literal["user", "assistant"]
    text: str
    ts: str

@dataclass
class Session:
    session_id: str
    project_path: str
    started_at: str
    sha256: str
    messages: list[Message]

@dataclass
class ScanCache:
    scope: Literal["global", "repo"]
    scanned_at: str  # ISO 8601 UTC
    sessions: list[Session]
```

Cache 파일 포맷(`.honne/cache/scan.json`)은 **기존과 동일** — bats 테스트 회귀 방지.

### 3.3 Interface/API Design

**계약 보존 원칙**: 기존 bash 스크립트의 공개 CLI args(`--input`, `--out`, `--scope`, `--since`, `--cache`, `--index-ref`, `--base-dir`, `--tags`, `--tag`, `--type`, `--types`, `--axis`, `--claim`, `--support-count`, `--prior-id`, `--quotes-file`, `--quotes-json`, `--top`, `--min-sessions`, `--max`, `--all`, `--keep-assets`, `--force`, `--jsonl`, `--until`, `--redact-secrets`, `--no-redact-secrets`)는 **Python CLI에서 전부 수용**. shim은 `"$@"` 순수 passthrough. deprecation은 별도 PRD.

**공통 parent parser** (`honne_py/cli.py`) — 모든 서브커맨드에 공통으로 추가되는 args:
```
--base-dir PATH             # .honne 루트 override (테스트용)
--debug                     # stderr 상세 로그
```

scope/since/cache/input 등은 서브커맨드 **자체 parser**에서 기존 이름 그대로 선언(공통 parent에 강제하지 않음 — 계약 drift 방지).

**서브커맨드** (실제 bash 계약 1:1 매핑):
| Command | 교체 대상 | 실제 수용 args | 출력 |
|---|---|---|---|
| `scan` | scan-transcripts.sh | `--scope --since --cache --index-ref --redact-secrets/--no-redact-secrets` | `.honne/cache/scan.json` |
| `extract lexicon` | extract-lexicon.sh | `--input --out --top --min-sessions` | `.honne/cache/<axis>.json` |
| `extract {obsession,reaction,workflow,ritual,antipattern}` | (신규, 현재는 analyze.py ad-hoc) | `--input --out` (축별 옵션은 구현 시 결정) | `.honne/cache/<axis>.json` |
| `detect-recurrence` | detect-recurrence.sh | `--input --out --min-sessions` | stdout JSON |
| `evidence gather` | evidence-gather.sh | `--input --claim --out --max` | stdout |
| `index session` | index-session.sh | `--jsonl --out` | `.honne/cache/index.json` |
| `query assets` | query-assets.sh | `--base-dir --scope --since --until --tag --tags --type --types --out` | stdout |
| `record claim` | record-claim.sh | `--type --axis --scope --claim --out --support-count --prior-id --quotes-file --quotes-json` | `.honne/assets/claim.jsonl` (append) |
| `purge` | purge.sh | `--all --keep-assets --force` | 삭제 |
| `precommit` | pre-commit.sh | (인자 없음) | exit 0/1 |

**Exit code 규약** (bats 호환):
- 0 success
- 1 invalid args
- 2 empty result
- 4 tool missing (python3 < 3.9)

### 4.3 State Management

- **Cache**: `.honne/cache/*.json` — atomic write via `tempfile.NamedTemporaryFile` + `os.replace` (기존 `mv $TMP $CACHE` 대체, atomicity 보장).
- **Assets**: `.honne/assets/*.jsonl` — append-only. 포맷 변경 없음.
- **Progress**: stderr로 `[scan] 100/604` 형태 출력. SKILL.md는 이를 Monitor로 tail — sleep polling 제거.

## 5. Edge Cases & Error Handling

| 시나리오 | 기대 동작 | 처리 전략 |
|---|---|---|
| python3 < 3.9 | shim 즉시 실패 | shim 맨 앞에서 `python3 -c 'import sys; sys.exit(0 if sys.version_info>=(3,9) else 4)'` 체크 |
| transcript JSONL 중간 line 파싱 실패 | 해당 line만 skip, 세션은 계속 | `json.JSONDecodeError` catch → stderr warn |
| `.message.content`가 string도 array도 아닌 타입 | 빈 text로 처리 | `isinstance` 분기 |
| 파일 0 bytes | 세션 skip | header 없으면 continue |
| sha256 index 충돌(동일 파일 재스캔) | skip | `_load_known_shas` set lookup |
| 604 이상 파일 | 문제 없음 | 스트리밍이라 메모리 상한 O(1 session) |
| 동시 실행(같은 cache 경로) | 마지막 쓰기 승자 | atomic replace로 corruption은 방지. lock은 out-of-scope |
| Ctrl+C 중단 | cache 미작성 | tmp 파일만 남음, 다음 실행 시 재시작 가능 |
| symlink loop | find가 따라가지 않음 | `Path.rglob`에 `follow_symlinks=False` (Python 3.13+) / 하위 호환은 `os.walk(followlinks=False)` |

## 7. Guardrails

### Project Guardrails (soft — warn on violation)

- shim bash 파일은 **3줄 이하** (shebang + version check + exec)로 제한. 로직은 Python에만.
- Python 코드에서 `subprocess.run(["jq", ...])` 호출 금지 — 순수 Python JSON 처리.
- 새 서브커맨드 추가 시 반드시 공통 parent parser 상속.
- stderr 진행률은 `[<cmd>] <n>/<total>` 포맷 통일 — SKILL.md Monitor 가이드가 이 포맷에 의존.

### Prohibitions (hard — stop immediately on violation)

- Cache JSON 스키마 breaking change 금지 (scope/scanned_at/sessions 키 및 session 하위 키). 변경 필요 시 별도 PRD.
- **CLI args breaking change 금지**: §3.3 표에 나열된 기존 bash args는 Python CLI가 전부 수용해야 함. arg 이름 변경·제거는 별도 deprecation PRD. shim은 `"$@"` 순수 passthrough 외 조작 금지.
- `_redact.py` / `_tokenize.py` 파일 자체 삭제 금지 — 테스트(`tests/unit_redact_test.py`, `tests/conftest.py`, `tests/unit_tokenize_test.py`)가 해당 경로에 의존. re-export shim으로만 축소 허용.
- `_redact.py`의 patterns set 축소 금지 — 보안 경로 regression.
- SKILL.md에 `sleep <N> && <cmd>` 패턴 재도입 금지.
- `.honne/cache/` 외부로 임시 파일 쓰기 금지 (`/tmp` ad-hoc 파일 안 됨 — 지금 로그에 있던 `.honne/scan_py.py` 같은 패턴 영구 봉쇄).
