## 1. Problem Definition

### 1.1 Current State

Haiku 4.5로 `/honne:whoami` 실행 시 4종 실패 관측 (실측 로그 기반):

| # | 증상 | 증거 |
|---|---|---|
| P1 | shim 비-0 exit 시 Claude가 "honne_py 모듈 누락"으로 오진, 사용자에게 설치 안내. 실제 스캔은 170세션 성공 | 로그 "No module named honne_py" 후 cd 재시도 성공 |
| P2 | python3 실행마다 승인 프롬프트. 리포 로컬 allow 패턴은 (a) honne 리포 밖에선 무효 (b) `PYTHONPATH=... exec python3` 형태여서 `Bash(python3 -m honne_py *)` 패턴 미매칭 | 사용자는 harnish 리포에서 실행, 승인 반복 |
| P3 | 리포트 증거 quote 0개, 축 제목+일반론만. evidence-gather/extract-lexicon 호출 누락, HITL에 "이런 게 있나요?"만 질문 | docs/honne.md 본문 "~있습니다" 6줄 |
| P4 | HITL·리포트에 한국어 환각("녹동", "맨듭니다", "맨른다인세성먼0멤른ꭉ데그나섘요") | SKILL.md에 고정 문구 없이 즉석 생성 |

공통 뿌리: SKILL.md가 문자열·판단을 자연어 지시로 모델에 위임.

### 1.2 Target State

- Haiku 4.5 종단간 완주, 승인 프롬프트 ≤ 2회
- `docs/honne.md` 각 축: ≥1 실제 transcript quote 또는 `[insufficient evidence]` 명시
- HITL·리포트 본문은 **스크립트 stdout을 echo**만 — 모델 생성 금지
- SKILL.md ≤ 90줄

### 1.3 Impact

저수준 모델에서 스킬 신뢰도 0. 플러그인 공개 후 "honne는 Opus 전용" 부정 평판 고착 위험.

## 2. Design Decision

### Alternative A: SKILL.md 지시 강화

- Pros: 변경 최소
- Cons: Haiku 추종률 낮음 (지금도 "HITL with quotes" 명시했으나 무시됨). SKILL.md 증가
- Reject: 결정론 불가

### Alternative B: 단일 CLI + HITL 블록 직접 방출 (선택)

- `scripts/honne` bash wrapper — python 호출·PYTHONPATH·에러 배너 일괄 처리
- `honne axis run <name> --locale <ko|en|jp> --emit-hitl-block` — 사용자에게 보일 **완성된 텍스트 블록**을 stdout에 방출. SKILL.md는 echo만
- `candidate_claim`은 **결정론적 추출**: top-k(=3) quote를 `templates/axes.<locale>.md`의 커넥티브로 연결. LLM 호출 없음
- 고정 문자열은 `templates/axes.{ko,en,jp}.md`, 금칙어는 `templates/forbidden.json`
- wrapper 비-0 종료 시 `templates/errors.txt`의 코드별 배너를 stderr에 방출. SKILL.md 규칙: "비-0 종료면 stdout+stderr를 해석 없이 전달 후 중단"

### Alternative C: Opus 전용 선언

- Reject: 플랫폼 자해, P2·P3는 모델 무관 구조 결함

### Selection: B

- **Rationale**: 4개 문제의 뿌리가 "자연어 지시 의존". 문자열·판단을 스크립트 stdout에 귀속시키면 모델 품질과 무관한 결정론 확보. 제약 "SKILL.md 길이 축소" + "저수준 모델 결정론" 동시 충족.
- **Rejected**: A (지시 추종 의존), C (플랫폼 자해)
- **Validity conditions**: ① 모든 사용자가 Opus+ 합의 시 A 회귀 가능 ② Claude Code 플러그인 설치 시 allow 자동 등록 지원되면 P2 해법 단순화
- **Referenced assets**: 없음 (honne 리포 내 .harnish 부재)

### 4.2 Key Interfaces

```
scripts/honne <sub> [...]
  scan ...                                             # 기존 동작 유지
  axis list                                            # stdout: 6개 축명
  axis run <name> --locale {ko|en|jp} --scan <path>
                                     [--emit-hitl-block]
  axis validate --text <str> --locale {ko|en|jp}
  doctor
  record|query|purge ...                               # 기존
```

**JSON 계약** (machine 모드, `--emit-hitl-block` 미사용):
```json
{
  "axis": "lexicon",
  "quotes": [{"session_id":"...","text":"...","ts":"..."}],
  "candidate_claim": "<extractive: top-k quotes joined by template connectives, or null>",
  "insufficient_evidence": false
}
```

**불변식**: `quotes=[]` ⇔ `candidate_claim=null` ⇔ `insufficient_evidence=true`. `candidate_claim`은 LLM 생성 금지 — 추출 합성만.

**`--emit-hitl-block`** (text 모드): `templates/axes.<locale>.md`의 axis 섹션 + quote 리스트 + "y / n / edit" 푸터로 **이미 완성된 블록**을 stdout에 출력. SKILL.md는 이 블록을 수정 없이 echo.

**`axis validate --text`**: 입력 텍스트에 `forbidden.json` 항목 포함 또는 길이 < 8자면 exit 2. HITL `edit` 응답 시 `record-claim.sh` 호출 직전 필수 통과.

### 4.3 SKILL.md 축소 (목표 ≤ 90줄)

- 제거: 현 SKILL.md:83-141 "LLM Prompt Templates" (58줄) → `axis.py` 내부 상수
- 제거: 현 SKILL.md:30-39 Monitor 예시 (10줄) → README
- 축소: Step 4 7줄 → 2줄 ("`honne axis list` → 각 축마다 `honne axis run <name> --emit-hitl-block`, stdout을 사용자에게 그대로 제시. `insufficient_evidence=true`면 HITL 스킵")
- 축소: Step 6 7줄 → 1줄 ("`honne axis run`이 이미 가드 통과한 본문 방출")
- 추가 규칙 1줄: "스크립트 비-0 종료 시 stdout+stderr를 해석·가공 없이 사용자에게 전달하고 중단"
- 산술: 140 − 58 − 10 − 6 − 6 + 1 ≈ **61줄**

### 4.4 Allow 패턴 옵션

user `~/.claude/settings.json` 에 택일:

(a) **엄격** (권장, 버전 업마다 재승인 필요):
```
"Bash(bash /Users/<you>/.claude/plugins/cache/honne/honne/*/scripts/honne *)"
```

(b) **편의** (범위 넓음, 보안 약화):
```
"Bash(bash * scripts/honne *)"
```

주의: `${CLAUDE_PLUGIN_ROOT}`는 settings.json에서 전개되지 않음 — literal 경로 사용.

### 4.5 exit code 표준 (doctor / wrapper)

| code | 의미 |
|---|---|
| 0 | ok |
| 2 | usage error |
| 66 | no input — 트랜스크립트 없음 (EX_NOINPUT) |
| 71 | python3 미탐색 (EX_OSERR) |
| 73 | `.honne/` 쓰기 불가 (EX_CANTCREAT) |
| 그 외 | 원인 미분류 — stderr 원문을 사용자에게 전달 |

SKILL.md는 코드 해석 시도 금지. wrapper가 코드별 배너 방출 책임.

## 7. Guardrails

### Soft (warn)

- SKILL.md 줄 수 증가 PR은 설명에 근거
- `templates/axes.*.md` / `forbidden.json` locale 동기화 — pre-commit 키 비교

### Hard (stop)

- `axis run`이 `quotes=[]` 이면서 `insufficient_evidence=false` 방출 금지 (계약 테스트 강제)
- `candidate_claim`을 LLM 호출로 생성 금지 — 추출 합성만
- SKILL.md에 한국어/일본어 고유 문구 하드코딩 금지 — 모두 `templates/` 경유
- wrapper 비-0 exit 의미를 SKILL.md·Claude가 해석·추측 금지
- Opus 전용 선언 금지 — Haiku 내결함성이 본 PRD 목적
