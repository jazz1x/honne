**목표**: bash+jq 기반 honne를 단일 Python entrypoint(`python3 -m honne_py`) + bash shim으로 재구성. scan 수십 초 이내 완료, CLI 계약 통일, 권한 prompt 최소화, bats+pytest 회귀 0.

- 기존 bats/pytest 전체 통과

**회귀 보호 원칙**:

#### 4.2.1 6축 extract 알고리즘 명세

각 함수는 scan.json을 읽어 축별 카운터/샘플을 집계 후 `{axis, counters, top_examples, session_coverage}` 형태로 `.honne/cache/<axis>.json`에 쓴다. 모든 정규식은 **대소문자 무시**, 매칭 대상은 `message.role == "user"`의 text.

```python
# honne_py/extract.py
def extract_lexicon(input: Path, out: Path, top: int = 50, min_sessions: int = 3) -> int:
    """word frequency over user messages.
    _tokenize로 token 분해 → Counter. min_sessions 이상 서로 다른 세션에서 등장한 word만 채택.
    상위 top개 단어 + 각 단어의 출현 세션 수."""

def extract_obsession(input: Path, out: Path) -> int:
    """canonical preamble 반복 패턴 + 언어 분리 감지.
    - preamble: 세션 첫 user 메시지의 10줄 해시. ≥50회 반복되는 해시를 preamble로 인정.
    - language split: 각 세션의 user 메시지 전체를 unicodedata로 한/영 문자 비율 계산.
      ko_ratio >= 0.3 → 'ko', else 'en'. ko/en 세션 수와 각 그룹의 평균 메시지 수 보고."""

def extract_reaction(input: Path, out: Path) -> int:
    """user의 assistant 출력 대응 패턴 — 4 카테고리 카운터.
    정규식(소문자 매칭, 한 메시지가 여러 카테고리 매칭 시 전부 카운트):
    - correction:   r'(아님|아니|틀렸|잘못|wrong|incorrect|not (right|correct))'
    - scope_cut:    r'(만|그만|stop|only|just|대신|하지마|skip)'
    - deepen:       r'(더|추가|계속|확장|deeper|more|continue|expand|elaborate)'
    - approval:     r'(좋|맞|맞아|완벽|good|perfect|correct|nice|yes[.!]?\s*$)'"""

def extract_workflow(input: Path, out: Path) -> int:
    """작업 시작 스타일 — 4 카운터(세션 단위가 아닌 user 메시지 단위).
    - code_first:        메시지 첫 2줄 안에 코드펜스 ``` 또는 경로(/./, `.py`, `.ts` 등) 포함
    - question_driven:   메시지가 '?' 또는 /(어떻게|왜|what|why|how)/로 시작
    - multi_file_scope:  메시지에 서로 다른 경로 ≥2개
    - small_targeted:    메시지에 단일 경로 + 함수명 또는 line 번호 참조"""

def extract_ritual(input: Path, out: Path) -> int:
    """세션 오프닝 스타일 — 세션별 첫 user 메시지를 3 카테고리로 분류(우선순위: direct_command > question > task_description).
    - direct_command:    r'^(해봐|실행|run|do|execute|make|build)\b'
    - question:          메시지가 '?'로 끝나거나 의문사로 시작
    - task_description:  위 둘 다 아닌 평서문 (기본값)
    각 카테고리 세션 수와 비율 보고."""

def extract_antipattern(input: Path, out: Path) -> int:
    """요청 품질 안티패턴 — 2 카운터.
    - overspec:  메시지에 r'(반드시|무조건|정확히|exactly|must|ALWAYS|NEVER|strictly|in this exact order)' 매칭 건수
    - repeat_same_request: 같은 세션 내에서 user 메시지 중 정규화(공백 압축, 소문자)된 텍스트가 2회 이상 등장하는 경우 카운트"""
```

**출력 스키마 공통** (`.honne/cache/<axis>.json`) — 본 정의를 v0.1.0 기준으로 **동결**. §7 Hard guardrail "Cache JSON 스키마 breaking change 금지"가 축별 cache에도 동일 적용:
```json
{
  "axis": "reaction",
  "generated_at": "ISO8601",
  "counters": { "correction": 407, "scope_cut": 400, "deepen": 417, "approval": 99 },
  "top_examples": [ {"session_id": "...", "snippet": "..."} ],
  "session_coverage": { "total_sessions": 578, "matched_sessions": 540 }
}
```

### 4.3 shim 표준 템플릿 (≤4줄)

```bash
#!/usr/bin/env bash
# scripts/<cmd>.sh — shim → python3 -m honne_py <cmd> "$@"
command -v python3 >/dev/null || { echo "python3 required" >&2; exit 4; }
exec python3 -m honne_py <cmd> "$@"
```

python3 부재 체크 1줄을 허용해 bash exit 127 대신 규약된 exit 4 반환. 그 외 로직 금지 — 모든 계약은 Python CLI가 소유.

## 5. Edge Cases

| 시나리오 | 기대 동작 | 처리 위치 |
|---|---|---|
| Python < 3.9 | shim 또는 cli.py가 즉시 exit 4, stderr "python3≥3.9 required" | F3 cli.py |
| `scripts/_redact.py` import 경로 누락 | `scripts/_redact.py`가 `sys.path.insert(0, Path(__file__).parent)` 후 `from honne_py.redact import *` | F7 |
| `scripts/_tokenize.py`가 subprocess 실행 대상 | `#!/usr/bin/env python3` + `python3 -m honne_py.tokenize_text` 위임으로 stdin/stdout 보존 | F8 |
| transcript JSONL line 손상 | 해당 line skip, stderr warn, 세션 계속 | F9 |
| `.message.content` 타입 예외(array/string 외) | 빈 text | F9 |
| 대용량 세션(>10MB) | 메시지 배열 누적 O(n), 세션 단위 스트리밍으로 프로세스 메모리는 O(max-session) | F9 |
| 동시 실행 같은 cache 경로 | 마지막 atomic replace 승자 (locking 없음) | F4 |
| Ctrl+C 중단 | 임시 파일만 남음, cache 미갱신 | F4 |
| shim 호출 시 python3 부재 | §4.3 표준 템플릿(≤4줄)의 `command -v python3` 체크가 stderr 메시지와 exit 4를 반환 — bash exit 127 회피 | F10~F26 |
| `.claude/settings.json` 기존 충돌 | `settings.local.json`만 있음 — 신규 생성이므로 충돌 없음. merge 로직 불필요 | F27 |

## 7. Guardrails

### Soft (위반 시 경고)
- shim `.sh`는 **4줄 이하** (shebang + 주석 + (선택)python3 존재 체크 + exec)
- stderr 진행률 포맷 `[<cmd>] N/M` 통일 — SKILL.md Monitor 가이드가 의존
- 새 서브커맨드 추가 시 `honne_py/__main__.py` subparsers에 등록 + `tests/unit_cli_contract_test.py`에 args 케이스 추가

### Hard (위반 시 즉시 중단)
- upstream §7 Prohibitions 전승: scan.json 스키마 breaking change 금지 / CLI args breaking change 금지 / `_redact.py`·`_tokenize.py` 삭제 금지 / redact 패턴 축소 금지 / SKILL.md에 `sleep N && <cmd>` 재도입 금지 / `.honne/cache/` 외부 임시 파일 금지
- Python 코드에 `subprocess.run(["jq", ...])` 금지 — 순수 Python JSON
