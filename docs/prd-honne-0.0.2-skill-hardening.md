## §1 Problem Definition

### 1.1 Current State (As-Is)

Real e2e execution of `/honne:whoami` (0.0.1) revealed four runtime failures and one silent data loss path. Additionally, the `/honne:persona` implementation (0.0.2) shipped with an untested branch and unverified synthesis quality.

**Runtime failures (e2e log)**

| # | Symptom | Root cause |
|---|---------|------------|
| 1 | 21 permission prompts during `/honne:whoami` (7 axes × ~3 bash calls) | No `allowedTools` pre-registration; every Bash call hits interactive approval |
| 2 | `record claim --quotes-json "$QUOTES_JSON"` → Exit 1, traceback on `reaction` axis | Korean + special chars in JSON arg → shell word-splitting / quoting failure at ARG_MAX boundary |
| 3 | Haiku wrote axis JSON to `/tmp/axis_reaction.json` (SKILL.md violation) | "Do NOT write to /tmp" instruction insufficient; Haiku still fell back to disk on long content |
| 4 | LLM narrative written to `~/.claude/projects/*/honne_synthesis.json` instead of `{PWD}/.honne/cache/narrative.json` | `{PWD}` in SKILL.md instruction is a prose placeholder — Write tool requires a resolved absolute path |

**Production-readiness gaps**

| # | Gap | Impact |
|---|-----|--------|
| 5 | `persona_no_axes.json` fixture (both antipattern + signature null) missing | `build_conflict_payload` both-null branch untested; regression risk |
| 6 | Synthesis template never passed a real LLM response | Hand-crafted `synthesis_full.json` fixture; actual LLM output format unverified |
| 7 | No e2e smoke procedure for `/honne:persona` | Feature shipped with zero end-to-end run |

### 1.2 Target State (To-Be)

1. `/honne:setup` outputs the exact `allowedTools` JSON fragment for `~/.claude/settings.json` — user pastes once, no further prompts
2. `record claim` accepts `--quotes-file <path>` (file-based JSON) alongside existing `--quotes-json`; SKILL.md Step 4 writes quotes to `.honne/cache/axis-{name}.json` and passes `--quotes-file`
3. SKILL.md Step 4 explicitly writes each axis output to `.honne/cache/axis-{name}.json`; `/tmp` prohibition is reinforced as a hard exit contract
4. SKILL.md Step 5 uses `python3 -c "import os; print(os.getcwd())"` to resolve the Write path before calling the Write tool
5. `persona_no_axes.json` fixture + `test_render_with_both_axes_null` test coverage added
6. `synthesis_full.json` fixture annotated with the real JSON schema; a validation helper in `persona_prompt.py` asserts required keys
7. Manual e2e checklist added to `docs/e2e-persona.md`

## §2 Design Decision

### Issue 2 — Data Passing: `--quotes-file` vs base64

**Alternative A: base64 inline**
- Encode quotes JSON as base64 before passing to shell. Avoids word-splitting.
- Con: Requires decode at receiver; opaque in error logs; fails on ARG_MAX for large quote sets (same root cause).

**Alternative B: file-based `--quotes-file` (Selected)**
- Write quotes JSON to `.honne/cache/axis-{name}.json` before record call; pass `--quotes-file` flag.
- Con: Extra file per axis (7 files); cleaned by `purge --keep-assets`.
- Selected: ARG_MAX failure was already observed in existing record call. File-based passing is the established pattern in this codebase (`scan.json`, `narrative.json`). `--quotes-json` preserved for backward compat (direct callers, tests).

**Validity condition**: revisit if `.honne/cache/` write fails on read-only mounts.

### Issue 1 — Permission: Setup Skill vs In-plugin allowedTools

**Alternative A: `/honne:setup` outputs settings fragment (Selected)**
- Skill reads `~/.claude/settings.json`, builds allowedTools diff, outputs it. User copies.
- Doesn't mutate user settings programmatically.

**Alternative B: Programmatic injection**
- Skill patches `~/.claude/settings.json` directly via Bash.
- Con: Modifies user-global state without explicit confirmation; hard to undo; violates "no CLAUDE.md injection" spirit.

Selected: User-visible, copy-pasteable, auditable. Programmatic mutation of `~/.claude/settings.json` is prohibited.

## §7 Guardrails

1. **`--quotes-file` must not break existing `--quotes-json` callers.** Both flags coexist. `--quotes-json` is deprecated but not removed. Tests cover both paths.

2. **`/honne:setup` must not write to `~/.claude/settings.json`.** Output only. User owns the mutation.

3. **SKILL.md `/tmp` prohibition must be testable.** `unit_persona_skill_test.py` already checks for `/tmp` strings in bash blocks — extend that check to `skills/whoami/SKILL.md`.

4. **`axis-{name}.json` cache files are ephemeral.** `purge --keep-assets` must delete them (they are not long-term assets). Add to the loop exclusion logic: keep `assets/` only, not `cache/axis-*.json`.

5. **synthesis JSON validation is a fail-fast guard, not a retry.** If required keys are missing, return 66 immediately. Do not attempt partial render.

6. **`persona_no_axes.json` fixture must use `null` (JSON null), not missing keys.** The code path branches on `axis is None` after `axes.get()` — both null and missing key must behave identically. Fixture uses explicit null to document the intended contract.
